# Iconografia e Tags

## Icones do menu backend

O registry backend usa nomes de icones estaveis para o frontend resolver com a biblioteca visual vigente.

Icones iniciais:

- `LayoutDashboard`: dashboards Platform/Tenant.
- `Building2`: Tenants.
- `Receipt`: Billing Cloud.
- `ShieldCheck`: Auditoria Platform.
- `ClipboardCheck`: W02A Checklists.
- `Settings`: Configuracoes.
- `UsersRound`: Usuarios.
- `ScrollText`: Auditoria tenant.
- `Bell`: Notificacoes.
- `ClipboardList`: Checklists operacionais.
- `PlayCircle`: executar checklist.
- `Wrench`: ordens de servico.
- `Map`: mapa operacional.
- `MapPin`: operadores em campo.
- `Route`: despachos e rotas.
- `Truck`: logistica.
- `MapPinned`: mapa logistico.
- `ChartNoAxesCombined`: financeiro.
- `ReceiptText`: cobrancas.
- `FileText`: faturas.
- `CreditCard`: pagamentos.

## Tags de status

Status oficiais de navegacao:

- `implemented`
- `partial`
- `mock`
- `planned`
- `backend-ready`
- `frontend-ready`
- `future`

Essas tags documentam maturidade da tela, mas nao autorizam acesso. A regra de exibicao continua sendo RBAC, boundary e modulo habilitado.
