Catálogo de Telas & Endpoints

ERP Techsolutions · Web (React) + Mobile (Flutter) · v2026.06

Inventário completo dos protótipos navegáveis: cada tela com seus componentes principais e o contrato de dados — entradas (leitura) e saídas (escrita). Web: 37 telas em 5 papéis · Mobile: 37 telas em 2 papéis e 2 tipos de atendimento (guincho / prestador).

Entradas → GET leitura/listagem Saídas → POST PUT/PATCH DEL escrita WS tempo real (WebSocket) ● já referenciado no protótipo (data-endpoint) ○ contrato REST proposto (validar com o time)

Convenções de API

AspectoPadrão
Base URLhttps://api.techsolutions.com.br/api/v1 · JSON · datas ISO-8601 · moeda em centavos (int)
AutenticaçãoPOST /auth/login → JWT (access+refresh). Header Authorization: Bearer <token> em toda chamada.
Multi-organizaçãoOrganização resolvida após login. Header X-Tenant-Id obrigatório. Permissões por papel (Gestor · Financeiro · Admin · Operação de Campo · Admin Plataforma).
Paginação / filtro?page=&pageSize=&sort=&q= + filtros por recurso. Listas grandes: { data, page, total }.
MutaçõesIdempotency-Key em POST (evita duplo-clique). Erros em RFC-7807 { type, title, detail }.
Sync (mobile)Escrita offline em fila local → POST /mobile/sync/batch. Conflito devolve 409 com as duas versões; resolução manual.
Tempo realWS /ws — canais: localização de campo, console de despacho, chat, notificações.

Endpoints ● aparecem como data-endpoint no markup; os demais (○) são o contrato sugerido, derivado das mesmas convenções.
